# Roster, Build Plan

> **Status:** Plan. Not yet scaffolded. See "Next session: where to start" at the bottom.

A reference Angular application built with the `angular-material-developer` skill. The point is not the app, the point is that **building this proves the skill is correct**. Every Angular Material component the skill documents gets used here; anything that doesn't match docs → fix the skill, then keep going.

## What it is

**Roster**, a small HR / people-management workspace for a fictional 200-person company. Employee directory, onboarding wizard, time-off requests, performance review cycles, and team / role settings.

The narrative fits Angular Material's strengths exactly. Material's two moats over Spartan, PrimeNG, and NG-ZORRO are **`<mat-form-field>` polish** and **`MatDatepicker`**, both of which HR tools lean on harder than any other domain. The directory exercises `MatTable` with sort + paginate + filter + virtual scroll. The onboarding stages exercise `MatStepper` plus CDK Drag-Drop. The shell exercises `MatSidenav` with the `BreakpointObserver` responsive pattern. M3 theming gets to shine with a custom brand-color palette generated by the `m3-theme` schematic.

## Goals

1. **Validate the skill end-to-end.** Use every Angular Material component the skill documents at least once. Wherever the skill is wrong, surface it now, not after publishing.
2. **Produce a portfolio-grade example.** Real interactions, real layouts, light + dark mode, keyboard nav, no console errors. Something we can point users at.
3. **Stay shippable in slices.** Each phase ends with a runnable app, no waterfall.

## Out of scope

- Auth (mock a logged-in HR admin)
- Real employee data backend (mock data in services, randomized within plausible ranges)
- SSR (later)
- Unit / E2E tests (this is manual visual + functional validation; the *skill* is what we're testing, not the app)
- Real email / Slack notifications (the time-off approval ends with a `MatSnackBar`, no outbound)
- Payroll / banking (the comp section is read-only mock numbers)

## Tech stack

- **Angular**, latest stable (v21+). Reactive Forms per project preference. Signal Forms moves in when Angular v22 ships stable.
- **Angular Material v21.2.14**, install via the skill's `setup.md` schematic (`ng add @angular/material`, also a meta-validation).
- **Material 3 theming** with a custom `roster` palette generated by `ng generate @angular/material:m3-theme`. Dark mode via `color-scheme: light dark` + a manual toggle.
- **Density `-1`** as the default (Material's `0` feels too spacious for HR-table contexts).
- **`@angular/cdk`** for Drag-Drop (onboarding kanban), VirtualScroll (directory), BreakpointObserver (responsive sidenav).
- **`@angular/material/datepicker`** with `provideNativeDateAdapter()`. Locale-aware via `MAT_DATE_LOCALE`.
- **Material Symbols Outlined** font for icons.
- **No additional state library**, signals + services only.
- **Zoneless**, opt in via `provideZonelessChangeDetection()`.

## Pages

### 1. Dashboard (`/`)
Landing snapshot. The first thing the HR admin sees on login.
- 4 KPI **Card**s with **Statistic**-style numbers: Active employees, Open positions, Pending time-off, Reviews due this cycle
- "On leave today" **List** with avatars + return-date
- "Recently joined" horizontal scroll of **Card**s (last 30 days hires)
- "Upcoming birthdays" timeline-ish list
- "Open onboarding tasks" mini-table
- **MatFab** at the bottom-right for quick "Invite employee"

### 2. People (`/people`)
The flagship table page.
- **Filters bar**: name search (**MatAutocomplete**), department **MatSelect**, status **MatButtonToggleGroup** (Active / On leave / All), team **MatSelect**, hire-date **MatDatepicker** range
- **MatTable** with: avatar + name + role, department, manager, location, joined, last review, status. Sortable (multi via Shift-click), paginated, sticky header, frozen first column (name) on horizontal scroll
- Row click → navigate to `/people/:id` (detail page)
- Trailing column with a **MatMenu** (3-dot): View profile, Send message, Edit, Archive
- Bulk select via header + per-row checkboxes using `SelectionModel<Employee>` from CDK
- **MatPaginator** at the bottom with page-size options
- "Invite employee" button → opens `/onboarding/new` route (a Stepper, see Page 4)
- Virtual-scroll variant toggle (**MatSlideToggle** "Compact mode") that swaps the table for a `cdk-virtual-scroll-viewport` of compact rows for >100 employees

### 3. Employee profile (`/people/:id`)
Tabs-driven detail page.
- **MatToolbar** header with breadcrumb, name + role + status **MatChip**, action buttons (Edit, Message, More via **MatMenu**)
- **MatTabGroup** with tabs:
  - **Overview**: Photo + summary, "About" form fields (read-only by default, Edit toggles to inputs), reporting line via **MatTree** (manager → reports), key metrics in **Card**s
  - **Compensation**: Salary history table, equity grants, benefits, "Adjust" action opens a **MatDialog**
  - **Reviews**: List of past review cycles, each with a score and a "View details" link. "Start review" button opens a Stepper-driven Dialog
  - **Documents**: File list with category tags, upload via a `MatInput[type=file]` styled with `mat-form-field`, **MatProgressBar** during simulated upload
  - **Activity**: **MatList** timeline of profile changes (role change, comp adjustment, doc upload, leave request)
- For sensitive employees (executives, flagged): a warning banner via **MatCard** with `color="warn"`-equivalent overrides

### 4. Onboarding (`/onboarding`, `/onboarding/new`, `/onboarding/:id`)
Three sub-routes:
- `/onboarding` → **Kanban board** with 4 columns (Offer / Setup / Day 1 / 30 days). Cards = new hires in progress. **CDK Drag-Drop** between columns updates the stage
- `/onboarding/new` → **MatStepper** (linear) creating a new hire:
  - Step 1: **Candidate** , name, email, role (**MatAutocomplete**), start date (**MatDatepicker**)
  - Step 2: **Team**, department (**MatSelect**), manager (**MatAutocomplete**), location, employment type (**MatRadioGroup**)
  - Step 3: **Equipment**, laptop model (**MatSelect**), accessories (**MatChipGrid** for "add a kit item"), shipping address (**MatInput**)
  - Step 4: **Review**, **MatDescriptions**-equivalent (a grid of label/value pairs using **MatCard**) plus a final confirmation toggle. Submit → **MatSnackBar** "Onboarding started for X"
- `/onboarding/:id` → per-hire checklist with progress (**MatProgressBar**), tasks split into stages, each task a `mat-checkbox` row, "Mark done" notes via **MatBottomSheet**

### 5. Time off (`/time-off`)
- **MatTabGroup** view switcher: Calendar / Requests / Balances
  - **Calendar**: full month view via a custom layout (no native Material calendar except inside Datepicker, so render with CSS grid + Material Card cells). Each day shows colored chips for people out. Click a date → **MatBottomSheet** with the people-out list
  - **Requests**: **MatTable** of all time-off requests with status filters. Each row has Approve / Reject actions (via the trailing menu). Approving a request fires a **MatSnackBar** with an Undo action
  - **Balances**: per-employee remaining time-off in a `mat-card` grid (vacation / sick / personal). Hover shows a **MatTooltip** with the accrual rate
- **MatFab** "New request" → **MatDialog** with a form: Employee (**MatAutocomplete**), Type (**MatSelect**), Date range (**mat-date-range-picker**), Reason (**MatInput** textarea), Submit
- Approval workflow: requests start `pending`, moving to `approved` / `rejected` via the row actions

### 6. Reviews (`/reviews`)
Performance review cycles.
- **MatExpansionPanel** accordion of past + current cycles: H1 2026, H2 2025, etc.
- Within a panel: per-employee status table (Self-assessment done / Manager review done / Calibration done / Final)
- Click an employee row → **MatDialog** with a **MatStepper** (Self → Manager → Calibration → Final), each step a form
- "Start new cycle" → wizard dialog with cycle name, dates, eligibility filters

### 7. Settings (`/settings`)
- **MatSidenav** within the page (not the global one) with these sections:
  - **General**: workspace name, time zone, week start day, default density (a **MatSlider** wired to global density override demo)
  - **Branding**: company logo (**Upload**), primary palette (**ColorPicker**-equivalent via swatches, or just **MatSelect** of M3 prebuilt palettes), preview region
  - **Teams**: list of teams with member count, click → drawer for editing
  - **Roles & permissions**: a `mat-list` of permissions, each a `mat-slide-toggle`, plus a "Per-role overrides" sub-section with **MatChipGrid** of role assignments
  - **Holidays**: per-month calendar of company holidays, click a day to toggle, plus an "Import country defaults" **MatSelect**
  - **Notifications**: **MatList** of preferences, each with a `mat-slide-toggle`

### 8. Not found (`/**`)
Catch-all. **MatCard** with a "404" icon, "Page not found" message, and a "Back to dashboard" button.

### Cross-cutting

- **Global ⌘K command palette** via a **MatDialog** with **MatAutocomplete**-style search for people, pages, actions
- **System notifications** via the header bell → a side **MatSidenav** opened in `mode="over"` showing a **MatList** of notifications with read/unread/archive
- **Toast feedback** via `MatSnackBar` (save success, copy-to-clipboard) and `MatDialog` for confirmation prompts
- **Confirmation on destructive actions** via `MatDialog.open(ConfirmDialog, {...})`
- **Dark mode toggle** in the header (sun / moon icon swap)
- **App shell**: `MatSidenavContainer` with `MatSidenav` (collapsible, responsive via `BreakpointObserver`) + `MatToolbar` + `<router-outlet>` inside `MatSidenavContent`

## Phases (shippable slices)

### Phase 0, scaffold + shell
- Angular CLI scaffold (`ng new roster --style=scss --skip-tests --ssr=false`)
- `ng add @angular/material` with custom theme, Roboto, animations
- `app.config.ts` providers: `provideZonelessChangeDetection`, `provideAnimationsAsync`, `provideRouter`, `provideNativeDateAdapter`
- M3 theme in `styles.scss` with `mat.theme(...)` using `mat.$blue-palette` to start (custom palette via schematic in Phase 7)
- App shell: `mat-sidenav-container` with collapsible `mat-sidenav` (router-linked `mat-nav-list`), `mat-toolbar` (logo, search trigger, theme toggle, notifications bell, avatar menu), `mat-sidenav-content` with `<router-outlet>`
- Route stubs for all 8 pages
- Dark mode runtime toggle via `<html>` class flip

### Phase 1, dashboard
- 4 KPI cards
- "On leave today" + "Recently joined" + "Upcoming birthdays" + "Open onboarding tasks" surfaces
- Mock data service (`MockData`), signal-based, hardcoded plausible data
- Skeleton states via `mat-progress-spinner` while the simulated 400ms load completes

### Phase 2, people directory
- Filters bar
- The flagship `MatTable` with sort, paginate, filter, sticky header, bulk select
- Trailing menu per row
- Compact mode toggle (swaps to CDK VirtualScroll)

### Phase 3, employee profile
- Tabs layout: Overview / Compensation / Reviews / Documents / Activity
- Edit mode toggle on Overview form
- Reporting line `MatTree`
- Dialogs for adjusting comp, uploading docs

### Phase 4, onboarding (kanban + new-hire wizard)
- CDK drag-drop kanban with 4 stages
- New-hire `MatStepper` (linear) with 4 steps + final review

### Phase 5, time off
- Tabbed view: Calendar / Requests / Balances

### Phase 6, reviews + settings

### Phase 7, polish + theming + deploy
- Run `ng generate @angular/material:m3-theme` with a brand color, swap the prebuilt palette for the custom one

- Empty states everywhere
- Loading states everywhere

## Mock data shape

A single `MockDataService` with signals exposes:
- `employees: WritableSignal<Employee[]>` (200 employees, varied roles/departments/locations)
- `teams: WritableSignal<Team[]>` (12 teams)
- `timeOffRequests: WritableSignal<TimeOffRequest[]>` (30 requests, mixed pending/approved/rejected)
- `reviewCycles: WritableSignal<ReviewCycle[]>` (4 cycles, current + 3 historical)
- `notifications: WritableSignal<SystemNotification[]>` (mixed read/unread)
- `holidays: WritableSignal<Holiday[]>` (12 standard US holidays + custom company days)

All derived state via `computed()`. Setters mutate the signals so the UI reacts.

5. Open `examples/roster/PLAN.md` (this file) for the Phase 0 shell checklist and start building.
