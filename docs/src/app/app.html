Anatomy

What's inside a skill

A skill is a folder of markdown. SKILL.md is a short router that tells the agent what's available and when to use it. The detail lives in topic-scoped reference files that get loaded on demand.

Why this shape

Routers and references, not one big file

An always-loaded SKILL.md stays under a few hundred lines. References are pulled in only for the task at hand, so context stays focused and the agent doesn't see (or repeat) patterns it isn't using.

FAQ

Questions, answered

Why per-library skills instead of one Angular UI skill? +

Angular UI libraries differ significantly in their architecture, theming model, and conventions. A single skill would either be a vague compromise or balloon past a usable context window. Specialized skills mean the agent activates the right one for your project and learns its specific idioms.

Do I need the angular-developer base skill? +

Strongly recommended. The base skill from Google covers Angular fundamentals: signals, dependency injection, routing, forms, SSR, accessibility. Our skills assume that foundation and focus on the component library on top.

Which agent runtimes are supported? +

Anything that loads markdown-based skills. The CLI ships adapters for 14+ runtimes including Claude Code, Cursor, Codex, GitHub Copilot, Gemini CLI, Continue, Cline, Roo Code, and more. You can also drop the skill folder into any agent that reads a skills directory.

How do you keep skills from drifting from the upstream? +

Every skill is paired with a reference example app. When the upstream library releases, we update the example, fix anything that broke, and fold the findings back into the skill. The example builds and runs, or the skill update doesn't land.

Can I contribute a skill for another library? +

Yes. Open an issue describing the library you want to cover. The pattern is one folder under skills/<library>-developer/ with a SKILL.md router plus topic-scoped references, plus a reference example app that exercises the surface.
